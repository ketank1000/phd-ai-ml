# 16. Software Engineering

> **Part B1 — Core Computer Science** | SIU PET Study Guide

---

- [ ] **SDLC Models**
  - **Waterfall**: Sequential, rigid, good for well-defined requirements
  - **Agile**: Iterative, flexible, customer collaboration (Scrum, Kanban)
  - **Spiral**: Risk-driven, combines waterfall + iterative
  - **V-Model**: Verification & validation at each stage
  - **Iterative/Incremental**: Build in iterations

- [ ] **Requirements Engineering**
  - Functional vs. non-functional requirements
  - SRS document (Software Requirements Specification)
  - Use cases, user stories

- [ ] **Design**
  - Design patterns: Singleton, Factory, Observer, Strategy, MVC
  - Cohesion (high is good) vs. Coupling (low is good)
  - Architectural styles: monolithic, microservices, layered, event-driven

- [ ] **UML Diagrams**
  - Structural: class diagram, object diagram, component diagram
  - Behavioral: use case, sequence, activity, state machine

- [ ] **Testing**
  - **Unit testing**: Individual components
  - **Integration testing**: Combined components (top-down, bottom-up, big bang)
  - **System testing**: Complete system
  - **Acceptance testing**: User validation
  - **Black-box**: Functional testing without code knowledge
  - **White-box**: Structural testing with code knowledge
  - Code coverage: statement, branch, path

- [ ] **Software Metrics & Quality**
  - Cyclomatic complexity: $V(G) = E - N + 2P$
  - Lines of code (LOC), function points
  - CMM/CMMI levels (1-5)
  - ISO 9001, ISO/IEC 25010 (quality model)
